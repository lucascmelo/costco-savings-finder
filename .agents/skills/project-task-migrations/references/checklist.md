# Migration Checklist

## Before drafting
- [ ] Read most recent /project/*.md file
- [ ] Extract current epics, tasks, and decisions
- [ ] Identify dependencies and constraints
- [ ] Confirm file name availability

## Before implementation
- [ ] Present context summary to user
- [ ] Show proposed file name and content
- [ ] Request explicit confirmation
- [ ] Verify user approval received

## Before completion
- [ ] Update task checkboxes as completed
- [ ] Append new decisions to decisions section
- [ ] Add discoveries to notes section
- [ ] Verify no historical files were modified

## Quality checks
- [ ] File name follows YYYY-MM-DD-topic.md format
- [ ] All tasks have actionable subtasks
- [ ] Decisions include rationale
- [ ] Context references previous migration
- [ ] No historical content was edited
